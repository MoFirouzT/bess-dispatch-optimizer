# Spec R1.1 — Deterministic MILP core

**Status:** Draft (awaiting human review)
**Release:** R1  **Depends on:** none (first build phase)

## Objective
A deterministic day-ahead dispatch MILP: given a known price vector and a battery spec, return the optimal grid-side charge/discharge schedule maximizing arbitrage revenue under SoC, power, mutual-exclusion, ramp, and terminal constraints.

## Formulation reference
Implements `docs/formulation.md` § "R1.1 — Deterministic core" in full. **No math outside that section.** Power is grid-side; efficiency is in the SoC balance only.

## Parameters / configuration
A `BatterySpec` config object (Pydantic v2 or dataclass) with defaults matching the sanity band (1-hour battery):

| Field | Default | Unit |
|---|---|---|
| `capacity` ($e_{\max}$) | 1.0 | MWh |
| `soc_min` ($e_{\min}$) | 0.0 | MWh |
| `p_charge_max` | 1.0 | MW |
| `p_discharge_max` | 1.0 | MW |
| `eta_charge` | 0.95 | — |
| `eta_discharge` | 0.95 | — |
| `ramp` ($R$) | `None` (disabled) | MW/period |
| `soc_initial` ($e_0$) | 0.0 | MWh |
| `soc_terminal` ($e^{\mathrm{tgt}}$) | 0.0 | MWh |

Prices and `dt` ($\Delta t$, default 1.0 h) are passed per solve, not stored on the spec. Oracles override efficiencies / ramp as noted.

## Golden oracles (exact expected values)
All use $T=3$, $\Delta t=1$, 1 MWh / 1 MW, $e_0=e^{\mathrm{tgt}}=0$, ramp disabled. Tolerance `1e-6`.

| # | inputs | expected objective | expected schedule | why this case |
|---|--------|--------------------|-------------------|---------------|
| 1 | $\pi=[10,50,20]$, $\eta=1$ | **40.0** | charge $[1,0,0]$; discharge $[0,1,0]$; soc $[1,0,0]$ | dispatch direction: buy low, sell high |
| 2 | $\pi=[10,50,20]$, $\eta^{ch}=\eta^{dis}=0.95$ | **35.125** | charge $[1,0,0]$; discharge $[0,0.9025,0]$; soc $[0.95,0,0]$ | efficiency placement (grid-side) |
| 3 | $\pi=[40,42,41]$, $\eta=0.95$ | **0.0** | all zero; soc $[0,0,0]$ | declines a trade whose spread < round-trip loss → idle |

**Derivation of oracle 2 (the one to check by hand):** charge $p^{ch}_1=1$ at price 10 → cost 10, $e_1=\eta^{ch}\cdot1=0.95$. To empty by $t_3$, discharge $p^{dis}_2$ with $p^{dis}_2/\eta^{dis}=0.95 \Rightarrow p^{dis}_2=0.9025$ at price 50 → revenue $45.125$. Objective $=45.125-10=35.125$.

**Note — discrepancy with the master plan.** The master plan's parenthetical "≈ 50·0.95 − 10/0.95 ≈ 36.97" is a loose *battery-side* approximation and is **not** the value under the canonical grid-side convention. The correct exact value is **35.125**. This is the formulation-review step doing its job; the spec value governs.

**Oracle 3 check:** best candidate trade buy@40 / sell@42 yields per-unit $0.9025\cdot42-40=-2.095<0$; every spread is loss-making after round-trip losses, so optimal is idle, objective 0.

## Property tests (invariants for any valid input)
For random valid price vectors and specs:
- **SoC bounds:** $e_{\min}\le e_t \le e_{\max}$ every $t$.
- **Power caps:** $0\le p^{ch}_t\le \bar P^{ch}$ and $0\le p^{dis}_t\le \bar P^{dis}$.
- **Mutual exclusion:** not ($p^{ch}_t>\epsilon$ and $p^{dis}_t>\epsilon$).
- **SoC continuity (exact):** $e_t \approx e_{t-1}+\eta^{ch}p^{ch}_t\Delta t - p^{dis}_t/\eta^{dis}\cdot\Delta t$.
- **Terminal:** $e_T \approx e^{\mathrm{tgt}}$.
- **Objective consistency:** solver objective $\approx \sum_t \pi_t \Delta t (p^{dis}_t-p^{ch}_t)$.
- **No phantom profit:** if every price is equal, objective $\le \epsilon$ (idle is optimal).
- **Ramp (when enabled):** $|p^{net}_t-p^{net}_{t-1}|\le R+\epsilon$.
