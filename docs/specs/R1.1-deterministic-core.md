# Spec R1.1 ‚Äî Deterministic MILP core

**Status:** Implemented (local gate green; CI pending first push)
**Release:** R1  **Depends on:** none (first build phase)

## Objective

A deterministic day-ahead dispatch MILP: given a known price vector and a battery spec, return the optimal grid-side charge/discharge schedule maximizing arbitrage revenue under SoC, power, mutual-exclusion, ramp, and terminal constraints.

## Formulation reference

Implements [formulation.md ¬ß "R1.1 ‚Äî Deterministic core"](../formulation.md#r11--deterministic-core) in full.
**No math outside that section.**
Power is grid-side; efficiency is in the SoC balance only.

## Parameters / configuration

A `BatterySpec` config object (Pydantic v2) with defaults matching the sanity band (1-hour battery):

| Field | Default | Unit |
| --- | --- | --- |
| `capacity` ($e_{\max}$) | 1.0 | MWh |
| `soc_min` ($e_{\min}/e_{\max}$) | 0.0 | p.u. |
| `p_charge_max` | 1.0 | MW |
| `p_discharge_max` | 1.0 | MW |
| `eta_charge` | 0.95 | ‚Äî |
| `eta_discharge` | 0.95 | ‚Äî |
| `ramp` ($R$) | `None` (disabled) | MW/period |
| `soc_initial` ($e_0/e_{\max}$) | 0.0 | p.u. |
| `soc_terminal` ($e^{\mathrm{tgt}}/e_{\max}$) | 0.0 | p.u. |

**SoC fields are per-unit** (fraction of `capacity`), per `docs/conventions.md` ¬ß2 and [ADR-0009](../decisions/0009-soc-per-unit-in-config.md): config is size-independent, the model is absolute MWh. The asset converts at registration ‚Äî $e_{\min}=\texttt{soc\_min}\cdot e_{\max}$, $e_0=\texttt{soc\_initial}\cdot e_{\max}$, $e^{\mathrm{tgt}}=\texttt{soc\_terminal}\cdot e_{\max}$ ‚Äî and `Schedule.soc` is reported in MWh. The full-charge bound is per-unit `1.0` (= `capacity`). Power/energy/capacity stay MW/MWh.

Prices and `dt` ($\Delta t$, default 1.0 h) are passed per solve, not stored on the spec. Oracles override efficiencies / ramp as noted.

## Golden oracles (exact expected values)

All use $\Delta t=1$, a 1 MWh / 1 MW battery (energy capacity / power rating ‚Äî a 1-hour, i.e. 1C, asset), $e_0=e^{\mathrm{tgt}}=0$, ramp disabled. $T=3$ unless noted.

**Tolerance `1e-6`.** Above HiGHS's numerical feasibility/optimality tolerance (~`1e-7`) and float round-off, yet far below any economically meaningful amount (sub-¬µ‚Ç¨). It absorbs solver noise while still catching real formulation errors ‚Äî it is *not* a knob to loosen when a test fails (CLAUDE.md ¬ß1).

| # | inputs | expected objective | expected schedule | why this case |
| --- | -------- | -------------------- | ------------------- | --------------- |
| 1 | $\pi=[10,50,20]$, $\eta=1$ | **40.0** | charge $[1,0,0]$; discharge $[0,1,0]$; soc $[1,0,0]$ | dispatch direction: buy low, sell high |
| 2 | $\pi=[10,50,20]$, $\eta^{ch}=\eta^{dis}=0.95$ | **35.125** | charge $[1,0,0]$; discharge $[0,0.9025,0]$; soc $[0.95,0,0]$ | efficiency placement (grid-side) |
| 3 | $\pi=[40,42,41]$, $\eta^{ch}=\eta^{dis}=0.95$ | **0.0** | all zero; soc $[0,0,0]$ | declines a trade whose spread < round-trip loss ‚Üí idle |
| 4 | $T=2$, $\pi=[-1,-1]$, $\eta^{ch}=\eta^{dis}=0.95$ | **0.0975** | charge $[1,0]$; discharge $[0,0.9025]$; soc $[0.95,0]$ | negative prices: battery profits as a *paid load*, not phantom profit |

**Derivation of oracle 2 (the one to check by hand):** charge $p^{ch}_1=1$ at price 10 ‚Üí cost 10, $e_1=\eta^{ch}\cdot1=0.95$. To empty by $t_3$, discharge $p^{dis}_2$ with $p^{dis}_2/\eta^{dis}=0.95 \Rightarrow p^{dis}_2=0.9025$ at price 50 ‚Üí revenue $45.125$. Objective $=45.125-10=35.125$.

**Oracle 3 check.** A cycle is profitable only if $\eta^{rt}\pi^{sell} > \pi^{buy}$ ‚Äî i.e. the price ratio must exceed $1/\eta^{rt} = 1/0.9025 \approx 1.108$ (a ~10.8% spread). The best pair here, buy@40 / sell@42, is only a 5% spread, so per grid-MWh cycled the profit is

$$\eta^{rt}\pi^{sell} - \pi^{buy} = 0.9025\cdot 42 - 40 = -2.095 < 0.$$

Every spread is loss-making after round-trip losses, so idle is optimal ‚Üí objective 0.

**Oracle 4 check (the negative-price subtlety).** With $e_0=e^{\mathrm{tgt}}$ and a single equal price $\pi$, energy balance forces $\sum_t p^{dis}_t = \eta^{rt}\sum_t p^{ch}_t$, so the objective collapses to $\pi(\sum p^{dis}-\sum p^{ch}) = \pi\,(\eta^{rt}-1)\sum p^{ch}$. For $\pi\ge 0$ this is $\le 0$ ‚áí idle is optimal (oracle 3's regime). For $\pi<0$ it is $\ge 0$ and the optimum charges at the cap: the battery is *paid* to consume the round-trip loss. Here $\pi=-1$, $\eta^{rt}=0.9025$: charge $p^{ch}_1=1$ (revenue $+1$), discharge $p^{dis}_2=0.9025$ (cost $0.9025$), objective $=1-0.9025=0.0975$. This is genuine ‚Äî not the simultaneous charge+discharge that the mutual-exclusion binary forbids.

## Property tests (invariants for any valid input)

For random valid price vectors and specs:

- **SoC bounds:** $e_{\min}\le e_t \le e_{\max}$ every $t$.
- **Power caps:** $0\le p^{ch}_t\le \bar P^{ch}$ and $0\le p^{dis}_t\le \bar P^{dis}$.
- **Mutual exclusion:** not ($p^{ch}_t>\epsilon$ and $p^{dis}_t>\epsilon$).
- **SoC continuity (exact):** $e_t \approx e_{t-1}+\eta^{ch}p^{ch}_t\Delta t - p^{dis}_t/\eta^{dis}\cdot\Delta t$.
- **Terminal:** $e_T \approx e^{\mathrm{tgt}}$.
- **Objective consistency:** solver objective $\approx \sum_t \pi_t \Delta t (p^{dis}_t-p^{ch}_t)$.
- **Objective floor:** objective $\ge -\epsilon$ always (idle is feasible, so the optimum is never negative).
- **No phantom profit:** if every price is equal **and $\ge 0$**, objective $\le \epsilon$ (idle is optimal). The bound is restricted to non-negative prices because under uniformly *negative* prices the battery legitimately profits as a paid load (consuming the round-trip loss at a negative price) ‚Äî see oracle 4.
- **Ramp (when enabled):** $|p^{net}_t-p^{net}_{t-1}|\le R+\epsilon$.
