# Spec R2.2b. Extreme-value (peaks-over-threshold) tail for the scenario bootstrap

**Status:** Implemented (gate green; five resolved decisions below, [ADR-0025](../decisions/0025-semiparametric-gpd-scenario-tail.md))
**Release:** R2  **Depends on:** R2.2 (the residual-path bootstrap this extends), R2.1c (fundamentals condition the point forecast the tail is added to; also the prerequisite for the *conditional* tail deferred below), R2.1 (residual history), R1.4a (walk-forward + leakage discipline)

## Objective

Give the R2.2 scenario generator a **parametric upper tail** so it can produce price spikes larger than any single historical forecast error. The residual-path bootstrap resamples whole-day residual vectors with replacement, so the largest spike any scenario can contain is capped at the historical maximum residual. R2.2b splices the residual **exceedances over a high threshold** with draws from a fitted **Generalized Pareto Distribution** (peaks-over-threshold), giving a semiparametric generator: empirical body (unchanged, preserving intra-day shape), extreme-value tail (un-capped). Opt-in; `tail=None` is byte-identical to R2.2.

## Motivation (the gap this closes)

For a day-ahead price series the **tails carry most of the decision-relevant information**: a scarcity spike is precisely the hour a battery wants to have reserved SoC to discharge into. The R2.2 residual bootstrap ([formulation В§R2.2](../formulation.md#r22-scenario-generation--reduction-uncertainty-representation-no-optimizer-change)) represents the *body* of the error distribution well but is **bounded above by history**: it can replay the worst spike it has seen, never a worse one, and it under-represents how often extreme excursions occur. So the R2.3 risk-aware program, and its CVaR tail estimate, are blind to any spike larger than the calibration window's maximum.

Extreme-value theory gives the standard fix. Above a high threshold `u`, exceedances of a broad class of distributions converge to a **Generalized Pareto** shape, so fitting a GPD to the residual exceedances and drawing tail excursions from it lets scenarios exceed the observed maximum in a principled, calibrated way, rather than by an ad-hoc multiplier. This is the scenario-layer analogue of what R2.1c did for the forecaster: it makes the uncertainty representation honest where it was previously blind.

**Why now:** R2.1c is the prerequisite. Spikes are residual-load-extreme events; with the point forecast now conditioned on residual load, the residual whose tail we model is a cleaner "surprise" signal, and the *conditional* tail (intensity as a function of residual load) becomes the natural follow-up (deferred below).

## Governing reference

- **Governing (the new theory; extreme-value / peaks-over-threshold):** S. Coles, *An Introduction to Statistical Modeling of Extreme Values* (Springer, 2001), chapters on threshold models and the Generalized Pareto Distribution. Authority for the POT limit, the GPD `(Оҫ, ОІ)` parameterization, threshold selection, and the return-level interpretation. Edition/chapter pinned and **verified before relying**.
- **Secondary (estimation, pointer only):** J. Hosking & J. Wallis, *Parameter and quantile estimation for the generalized Pareto distribution* (Technometrics, 1987) for the **probability-weighted-moments (PWM)** estimator (closed-form, so the fit is pure-numpy and deterministic, no `scipy`, mirroring R2.2's dependency decision 5).
- **Notation reconciliation:** house style wins. Residuals stay `r^(m) = y^(m) вҲ’ ОјМӮ^(m)` (вӮ¬/MWh, grid-side, the R2.2 definition); the tail acts on the residual, so a spliced spike is `ОјМӮ + (tail-augmented residual)`. GPD shape `Оҫ` and scale `ОІ`; threshold `u` set from a residual-magnitude quantile.
- **Alternatives considered:** *block-maxima / GEV* (fit the generalized extreme-value distribution to per-block maxima) is the classic EVT alternative to POT; rejected because block maxima waste the sub-maximal exceedances POT uses, and day-ahead residual history is short. *Ad-hoc spike multiplier* (scale the worst historical residual by a constant) is rejected: no calibration, no return-level meaning. *Full multivariate/extremal-dependence tail* (a joint tail over all `T` hours) is out of scope (see below): R2.2b splices per-component exceedances onto the empirical day-shape, borrowing the *which-hours-spike* structure from real vectors.

## Design sketch (subject to review)

- **Semiparametric splice, in place, same `n`.** Draw whole-day residual vectors exactly as R2.2 does (preserving intra-day shape and the equiprobable `p_s = 1/n`). Then, componentwise, where a drawn residual `r_t` exceeds the threshold `u`, **replace the excess** `(r_t вҲ’ u)` with a fresh draw from `GPD(Оҫ, ОІ)`; below `u` the value is untouched. The body is the empirical bootstrap; only the exceedances become parametric and can exceed history. Same scenario count, same probabilities, drop-in `ScenarioSet`.
- **PWM fit, pure numpy (self-contained; verified against Hosking-Wallis 1987).** Fit `(Оҫ, ОІ)` to the pooled excesses `y = r вҲ’ u` (only exceedances) by probability-weighted moments (closed form, deterministic, no `scipy`). With `yвӮҚвӮҒвӮҺ вүӨ вҖҰ вүӨ yвӮҚвӮҷвӮҺ` the ascending order statistics: `aвӮҖ = mean(y)`, `aвӮҒ = (1/n) ОЈбөў ((nвҲ’i)/(nвҲ’1)) yвӮҚбөўвӮҺ`, then `ОҫМӮ = 2 вҲ’ aвӮҖ/(aвӮҖ вҲ’ 2aвӮҒ)` and `ОІМӮ = 2В·aвӮҖВ·aвӮҒ/(aвӮҖ вҲ’ 2aвӮҒ)`. (Sanity: exponential excesses give `aвӮҒ = aвӮҖ/2В·(1/2)` вҮ’ `ОҫМӮ = 0`, `ОІМӮ = mean`, the memoryless case.) GPD tail CDF `H(y) = 1 вҲ’ (1 + Оҫy/ОІ)^(вҲ’1/Оҫ)` (`Оҫ вү  0`), `1 вҲ’ e^(вҲ’y/ОІ)` (`Оҫ = 0`); inverse-CDF sampling draws excess `y = (ОІ/Оҫ)[(1вҲ’U)^(вҲ’Оҫ) вҲ’ 1]` for a uniform `U`.
- **Upper tail primary; lower tail symmetric.** The decision-relevant spike for storage is the *upside* (discharge into scarcity), so the upper residual tail is modeled first. Negative-price spikes (a charging opportunity) are the symmetric lower-tail extension, posed as an open question, not built by default.

## Golden oracles

The GPD fit and the splice are exact arithmetic, so R2.2b gets real golden oracles, not only statistical gates.

| # | inputs | expected | why this case |
|---|--------|----------|---------------|
| 1 | `tail=None` | scenario set byte-identical to R2.2 `generate_scenarios` | opt-in identity: off вҮ’ exactly R2.2 |
| 2 | threshold `u` above every residual (no exceedances) | identical to the plain bootstrap | degenerate tail is a strict extension, no exceedances to splice |
| 3 | fixed residual sample | PWM `(Оҫ, ОІ)` equal a hand-computed value | pins the estimator arithmetic |
| 4 | one residual component above `u`, fixed seed, known `(Оҫ, ОІ)` | body components untouched; the exceedance equals `u + GPDвҒ»В№(draw)` | pins the splice: only the excess is replaced, arithmetic exact |

## Property tests

- **Opt-in identity:** `tail=None` вҮ’ bit-identical `ScenarioSet` to R2.2 on the same seed/inputs.
- **Body preserved:** below-threshold residual components are unchanged by the splice (the tail touches only exceedances).
- **Un-capping:** with `Оҫ > 0`, some generated path exceeds the historical maximum residual (the defect is fixed); with `Оҫ = 0` the tail is exponential and still un-caps.
- **Valid measure:** `probs вүҘ 0`, `ОЈ probs = 1`; scenario count unchanged by augmentation.
- **Tail-heaviness monotone:** larger `Оҫ` (or larger `ОІ`) yields a stochastically heavier upper tail (larger expected path maximum) over fixed seeds.
- **Determinism:** fixed seed + fixed inputs + fixed `TailModel` вҮ’ bit-stable `ScenarioSet`.

## Out of scope

- **Conditional tail (residual-load-dependent GPD).** The richest target R2.1c unblocks: threshold or `(Оҫ, ОІ)` as a function of residual load, so spikes are predicted on low-renewable / tight-margin days. Deferred (data-hungry: extremes are rare and splitting by covariate makes them rarer) as R2.2c / named future work; the dependency ordering (fundamentals вҶ’ unconditional tail вҶ’ conditional tail) is on record.
- **Multivariate / extremal-dependence tail.** A joint tail over all `T` hours (how spikes co-occur across hours). R2.2b splices per-component onto the empirical day-shape, borrowing co-occurrence structure from real vectors instead of modeling it.
- **Block-maxima / GEV**, ad-hoc multipliers: rejected alternatives (see governing reference).
- **Any optimizer / R2.3 change.** The tail-augmented set is a drop-in `ScenarioSet`; the reduction and the two-stage program are untouched.
- **Reduction re-design** beyond the tail-preservation question posed below.

## Open questions

Phase-local decisions only; posed with a proposed answer, resolved in place at review (2026-07-24). Resolved lines kept as the decision trail.

- **Does scenario reduction destroy the tail we just added?** Forward selection minimizes Kantorovich distance, and rare extreme paths are far from the mass, so R2.2's reducer may drop exactly the spike scenarios R2.2b creates. *Proposed:* measure it first; if the tail is lost, reduce body and tail separately. **Resolved (2026-07-24): measure during implementation, act on the measurement.** A property test checks whether the reduced set retains a path above the historical-max residual; if reduction destroys the tail, add a tail quota (reduce the extreme subset separately from the body) and pin it with that test. If reduction already keeps a representative extreme, no quota is added and the measurement is recorded. This is the one real interaction with existing R2.2 code, so it is settled empirically, not by assumption.
- **Threshold selection: fixed quantile or diagnostic-driven?** *Proposed:* fixed `0.95` + a diagnostic plot. **Resolved (2026-07-24): fixed `0.95` residual-magnitude quantile** as the shipped default (simple, reproducible), with the mean-residual-life / parameter-stability plot in the example figure as the *diagnostic* justifying it. Automated threshold selection stays out of scope.
- **Upper tail only, or lower tail too?** *Proposed:* upper first. **Resolved (2026-07-24): ship `"upper"`** (the storage-relevant discharge spike); `"lower"` (negative-price charging spike) is the symmetric extension, built only if the tail-calibration gate shows the lower tail is also capped in a decision-relevant way. Keeps the first cut contained; the `TailModel.side` field leaves room for it.
- **Phase id / naming.** *Proposed:* R2.2b. **Resolved (2026-07-24): R2.2b**, a scenario-generation extension parallel to R2.1b (a forecaster extension); the conditional tail becomes R2.2c if built.
- **Governing reference under the textbook-first policy.** *Proposed:* Coles (2001) governing. **Resolved (2026-07-24): Coles, *An Introduction to Statistical Modeling of Extreme Values* (Springer, 2001) governing** for the POT/GPD theory (a textbook and the standard EVT reference, so no policy deviation, unlike R2.2's paper-defined reduction method), with Hosking & Wallis (1987) subordinate for the PWM estimator only. Chapter/section verified before relying (recorded in `references.md` В§R2.2b).
