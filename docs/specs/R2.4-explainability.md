# Spec R2.4. Shadow-price explainability

**Status:** Implemented (gate green, 2026-07-16; six decisions resolved, formulation §R2.4 + ADR-0023 written)
**Release:** R2  **Depends on:** R1.1 (the MILP whose duals these are), R1.2 (degradation enters the no-trade band, and the per-period cost this phase surfaces), R1.5 (the serving surface the endpoint extends)

## Objective

Surface the dual of the state-of-charge balance, the marginal **water value** of stored energy, so a trader reading a schedule can see *why* the battery charged, discharged, or sat idle through a price spike.
The deliverable is a per-period water value in EUR/MWh, the no-trade band it implies, a per-trade breakeven-slippage read-off from that band, and an endpoint that serves them.
This phase adds no new dispatch decision: it explains the one R1.1 already makes, and never re-optimises it.

## The design driver: a MILP has no duals

The dispatch is a MILP (the mutual-exclusion binary $u_t$, R1.1 constraint (3)).
Integer programs have no dual in the LP sense, so there is no shadow price to read off the solved model.
The standard escape is **fix-and-resolve**: take the optimal commitment $u^\star$ from the MILP, fix it, and re-solve the resulting LP, whose duals are then well defined.

In its plain form the escape is sound in the objective.
Fixing $u = u^\star$ restricts the MILP's feasible set to a subset that still contains the MILP optimum, so the LP optimum equals the MILP optimum exactly, not approximately.
The duals it yields are valid for perturbations small enough not to change the optimal commitment, which is the honest caveat and the reason the reported value is conditional on $u^\star$.

**The trap is that $u^\star$ is not unique.**
At an idle period ($p^{ch}_t = p^{dis}_t = 0$) both $u_t = 0$ and $u_t = 1$ are optimal, and the solver's tie-break is arbitrary.
The tie-break is not cosmetic: constraint (3) gates each direction on $u_t$, so whichever value the solver returns clamps one direction's cap to zero, and that clamp carries a multiplier that changes the reported water value.
Three candidate rules for the re-solved LP each reproduce the MILP objective and each report a **different** water value:

| rule | the re-solved LP at an idle period |
| --- | --- |
| `fix_u` | keep $u^\star$ as the solver returned it |
| `free_idle` | relax both exclusion caps to the natural power caps $\bar P^{ch}, \bar P^{dis}$ |
| `fix_zero` | fix $p^{ch}_t = p^{dis}_t = 0$ and drop the gated caps |

Section "De-risked before drafting" measures the spread:
on the reference instance the three report **200 / 100 / 10** EUR/MWh while all three return the MILP objective 190.
Objective equality therefore does **not** discriminate them.
Resolved decision 1 records which rule ships and why; oracle 1 is the gate that holds it.

**Escaping the trap costs the soundness the plain rule had, so the guard is not optional.**
Only `free_idle` recovers the true marginal value, and it does so by *relaxing* rather than restricting, which forfeits the containment argument above and lets the LP beat the MILP on a measured class of instances.
The shipped rule therefore narrows the relaxation to where it is safe and asserts objective equality on every solve, so a dual the dispatch model does not support is raised rather than reported.

## Formulation reference

The delta below is added to `formulation.md` as section R2.4, as the first build task, ahead of code. It is derived from R1.1 and R1.2 and introduces no new decision variable, constraint, or objective term. Summarized here in house notation (grid-side power, per-unit SoC config, $\pi/e/\eta/\Delta t$).

**The water value.** Let $\mu_t$ (EUR/MWh) be the dual of the SoC balance, R1.1 constraint (1):

$$e_t = e_{t-1} + \eta^{ch} p^{ch}_t \Delta t - \frac{p^{dis}_t}{\eta^{dis}} \Delta t.$$

$\mu_t$ is the marginal value of one extra MWh of stored energy at the end of period $t$: the rate at which the optimal objective improves per MWh injected free into the balance. Equivalently $\mu_1 = \partial V^\star / \partial e_0$.

**Water value is flat while SoC is interior.** Stationarity in $e_t$ gives $\mu_t = \mu_{t+1} + \beta_t$, where $\beta_t$ is the multiplier on whichever SoC bound binds at $t$. So wherever $e_{\min} < e_t < e_{\max}$, the water value does not move: $\mu_t = \mu_{t+1}$. Stored energy carries its value forward unchanged, and the water value steps only where the battery hits a bound. This is what lets a single number explain a whole day.

**The no-trade band.** Stationarity in $p^{ch}_t$ and $p^{dis}_t$, with the R1.2 linear degradation cost $D_t = c^{deg} \tau_t$ on storage-side throughput $\tau_t = \eta^{ch} p^{ch}_t \Delta t + p^{dis}_t \Delta t / \eta^{dis}$, gives the sign conditions

$$p^{ch}_t > 0 \implies \pi_t \le \eta^{ch}\bigl(\mu_t - c^{deg}\bigr), \qquad p^{dis}_t > 0 \implies \pi_t \ge \frac{\mu_t + c^{deg}}{\eta^{dis}},$$

and therefore, at an idle period, the band

$$\boxed{ \eta^{ch}\bigl(\mu_t - c^{deg}\bigr) \le \pi_t \le \frac{\mu_t + c^{deg}}{\eta^{dis}} }$$

**This band is the explanation.** The battery sits idle exactly when the price falls inside it. Charging buys a MWh at $\pi_t$ and stores $\eta^{ch}$ of it, so it pays only when $\pi_t$ is below the water value net of wear and shrunk by charge loss. Discharging sells a MWh but drains $1/\eta^{dis}$, so it pays only above the water value grossed up by both. The band's width,

$$\mu_t\Bigl(\frac{1}{\eta^{dis}} - \eta^{ch}\Bigr) + c^{deg}\Bigl(\frac{1}{\eta^{dis}} + \eta^{ch}\Bigr),$$

is created by round-trip loss and wear, not by the price. At $\eta^{rt} = 1$ and $c^{deg} = 0$ it collapses to the point $\pi_t = \mu_t$, and idling means exact indifference. A trader asking "why did it sit through a 100 EUR spike" reads: the water value was 100, so the spike bought nothing.

**A transaction cost widens the band, and this is the hook for a robustness read (decision 4).** A slippage / fee $\kappa$ per grid-side MWh traded needs a larger spread to justify a trade, so the band becomes

$$\eta^{ch}\bigl(\mu_t - c^{deg}\bigr) - \kappa \le \pi_t \le \frac{\mu_t + c^{deg}}{\eta^{dis}} + \kappa.$$

Note $\kappa$ enters **flat on each side, outside the $\eta$ factors**, unlike the $\eta$-scaled wear, because it is a market fee on grid energy rather than storage-side throughput (the exact modeling, grid- vs storage-side and symmetric vs asymmetric, is fixed when this is derived into the formulation delta). This yields a per-trade **breakeven slippage**, the margin by which each executed trade clears its $\kappa = 0$ threshold ($\eta^{ch}(\mu_t - c^{deg}) - \pi_t$ for a charge, $\pi_t - (\mu_t + c^{deg})/\eta^{dis}$ for a discharge), which is $\ge 0$ by optimality and says how much cost the schedule can absorb before each trade flips. It is a read-off from the existing schedule and its duals, changing no objective term and re-solving nothing, so it touches no R1 oracle. Re-optimising *with* $\kappa$ in the objective is a different thing and out of scope (decision 4).

**Considered but out of scope:** duals of the R2.3 two-stage program (decision 3); parametric ranging over the price vector (how far $\pi_t$ can move before the commitment changes); Benders or L-shaped decomposition; bid-curve construction from the water value (Release 3).

## Design decisions

- **ADR-0023. The re-solve rule for MILP duals.**
Records the shipped rule (`free_idle` restricted to $\pi_t \ge 0$), the measured 200 / 100 / 10 spread that forces the choice, the closed-form dump that bounds where the relaxation is safe, and the objective-equality guard.

## De-risked before drafting

The central decision was measured, not reasoned about, so the spec proposes a rule that is already known to work.
The tables below come from several independent randomized runs at different seeds and instance counts, so the absolute count of fallback (negative-priced idle) periods differs between them (they are rare, so each run surfaces a different number); within each table it is the ratio, not the count, that the claim rests on.

**Reference instance.** $T = 3$, $\pi = [10, 100, 200]$, a 1 MW / 2 MWh battery, $e_0 = e^{tgt} = 0$, $\eta = 1$, ramp disabled, no degradation. The MILP charges at $t_1$, **idles at $t_2$ through the 100 spike**, discharges at $t_3$; objective 190. The 2 MWh capacity is deliberate: it keeps SoC strictly interior, so no bound multiplier muddies the reading.

**The true answer, computed independently.** By finite difference on the MILP itself, $\partial V^\star / \partial e_0 = 100$. The economics confirm it: a free MWh sitting in the battery at the start cannot be sold at $t_3$, where power is already capped, so it must clear at $t_2$, whose price is 100. The marginal MWh is worth 100, not 200.

**Measured, all three rules against that truth:**

| rule | re-solved LP objective | reported $\mu$ | matches $\partial V^\star/\partial e_0 = 100$ |
| --- | --- | --- | --- |
| `fix_u` | 190 | [200, 200, 200] | no |
| `free_idle` | 190 | [100, 100, 100] | **yes** |
| `fix_zero` | 190 | [10, 10, 10] | no |

Only `free_idle` recovers the true marginal value. `fix_u` and `fix_zero` return the ends of the band the tie-break leaves open, and both are defensible-looking numbers that are simply wrong.

**A randomized property test does not catch this.** Across 200 random instances (varying $\eta$, negative prices, degradation) all three rules matched the finite-difference truth. Random instances pin the water value through their trading periods, so the idle-rule ambiguity never binds. The property is not vacuous, it tests something real, but it cannot separate a correct rule from two wrong ones. The constructed instance above can, which is why oracle 1 carries the gate and no property test can replace it. A random draw is not a substitute for the case designed to hurt.

**`free_idle` is not sound as stated, and the failure has a closed form.** On 2 of 600 random instances the relaxed LP **beat** the MILP objective. Both violations sat at $\eta^{rt} < 1$ with a negative-priced idle period, and the excess matched

$$\text{gap} = \sum_{t \text{ idle},\ \pi_t < 0} |\pi_t| \bigl(1 - \eta^{ch}\eta^{dis}\bigr) \bar P \Delta t$$

exactly (observed +1.354275 and +12.9426, both reproduced to four decimals). The mechanism: with both directions freed, the LP charges and discharges at once, holding SoC flat while burning the round-trip loss, and a negative price pays it to do so. That action is physically real but R1.1's exclusion constraint forbids it, so it is outside the model and must not leak into the dual. At $\eta^{rt} = 1$ the round trip is exactly cash-neutral and energy-neutral, which is why no violation appears there.

**The restricted rule is sound and still correct.** Relaxing the exclusion cap only at idle periods with $\pi_t \ge 0$, and keeping $u^\star$ fixed at negative-priced idle periods, gives **0 objective-equality violations across 600 instances** while still returning $\mu = [100, 100, 100]$ on the reference instance. The fallback engaged on 12 periods across those 600 instances, all negative-priced.

**The two derived claims above were then measured, and one of them bounded the fallback's cost.** Across 500 randomized instances (varying both efficiencies independently, negative prices, degradation, both power caps):

| claim | checks | violations |
| --- | --- | --- |
| water value flat on interior SoC ($\mu_t = \mu_{t+1}$) | 1147 | **0** |
| no-trade band agrees with the action, at a **non-fallback** period (trading, or idle with $\pi_t \ge 0$) | 2503 | **0** |
| no-trade band agrees with the action, at a **fallback** period (idle with $\pi_t < 0$) | 21 | **13** |

So the band is exactly as good as the rule that produced it. Where the relaxation applies it is clean; at a negative-priced idle period the band is **wrong most of the time**, and an explanation that reports a band the action contradicts is worse than one that declines. This is what decision 2 resolves: not whether to trust every fallback band (clearly not), but whether any of them can be trusted.

**A hypothesis the data killed.** The natural guess is that the battery idles at a negative price because it is physically blocked, full, so the honest reason there is a binding SoC bound rather than a band. Measured over 35 such periods: only **2 (6%)** sit at $e_{\max}$, 18 sit at $e_{\min}$, and 15 sit strictly inside. Physical blocking is not the explanation, and the binding constraint is often intertemporal (a later discharge cap, or wear making the round trip unprofitable) rather than local. Do not re-derive this: the fallback's problem is the tie-break, not the physics.

**The rival rule, measured and rejected: intersect the two tie-breaks.**
Since $u_t$ is binary, the idle set admits exactly two tie-breaks, and each is sound on its own because neither frees both directions at one period and so neither can run the dump.
Solving both and intersecting looked like it would pin the value soundly, dissolving the $\pi_t \ge 0$ restriction and the fallback together.
**It does not work, and the reason is worth keeping.**
On the reference instance both tie-breaks return the correct objective 190 and report $\mu = 200$ ($u_t = 0$) and $\mu = 10$ ($u_t = 1$): the truth, 100, lies strictly between them and equals neither, so intersecting the reported values yields nothing and spanning them yields the useless $[10, 200]$.
A solver returns **a vertex of the dual optimal face, not the face**. The faces here really are $[100, 200]$ and $[10, 100]$ and really do intersect at $\{100\}$, but HiGHS hands back one vertex of each (the far ends), discarding exactly what the intersection needs. Recovering the faces means building the explicit dual and maximising and minimising $\mu_t$ per period per tie-break, roughly $4T$ auxiliary LPs.

**Why that rival is redundant anyway: `free_idle` is already the intersection, in one LP.**
The two reduced-cost conditions at an idle period invert to closed-form bounds in the price alone:

$$\pi_t \eta^{dis} - c^{deg} \le \mu_t \le \frac{\pi_t}{\eta^{ch}} + c^{deg}$$

Each tie-break can impose only one side, because it clamps one direction's cap to zero and destroys that direction's reduced-cost condition. `free_idle` frees both and imposes both at once, landing on the intersection directly. Measured over 400 instances and 205 idle periods:

| rule | the bound it can impose | violations |
| --- | --- | --- |
| tie-break $u_t = 0$ (clamps charge) | lower only, $\mu_t \ge \pi_t \eta^{dis} - c^{deg}$ | 0 / 205 |
| tie-break $u_t = 1$ (clamps discharge) | upper only, $\mu_t \le \pi_t / \eta^{ch} + c^{deg}$ | 0 / 205 |
| `free_idle` | **both** | 0 / 203 each |

So the rival is not a better rule, it is the same answer for $4T$ times the solves, and `free_idle`'s dump is the price of getting it in one: the relaxation that pins $\mu_t$ from both sides is the same relaxation that opens the primal to simultaneity.

**The fallback is a property of the flat run, not the period, and it is cheaply detectable.**
Because $\mu$ is constant while SoC is interior, a tie-break shifts the level of a whole **flat run** of periods rather than one period, so a run pinned by anything else is immune to it.
Measured over 110 flat runs: pinnedness is **constant within every run, 110 / 110**.
The mechanism is exact: a run containing a trading period dispatching **strictly between zero and its cap** is pinned **76 / 76**, because such a period's reduced cost is an equality rather than an inequality (a period at its cap gives only a one-sided bound, which is why the reference instance stays ambiguous).
The practical consequence: of 38 fallback periods, the tie-break did not matter for **17 (45%)**.

Detection needs no theory and no dual face: solve the fallback tie-break **both ways** and compare $\mu$. Where the two agree the tie-break never mattered, whatever the reason. That is **one extra LP** for the whole horizon, taken only when a fallback period exists, against $4T$ for the dual-face route. Both tie-breaks are sound in every instance measured (0 unsound), since neither frees both directions at one period.

**The decision table this produces settles what may be reported.** Over 81 fallback periods, cross-tabulating pinnedness against the closed-form band:

| pinned | band empty | band holds | count |
| --- | --- | --- | --- |
| **true** | false | **true** | **35** |
| false | false | false | 25 |
| false | true | false | 18 |
| false | false | true | 3 |

**No row has `pinned` true and the band failing**, so pinnedness is a sufficient condition to report the band, 35 / 35. Unpinned periods fail it 43 times in 46; the 3 that hold do so by luck and nothing distinguishes them, so they stay suppressed. No empty band ever holds, as required.

**The band inverts at a negative price, and degradation is what reopens it.** For $\pi_t < 0$ the lower edge $\pi_t \eta^{dis} - c^{deg}$ rises above the upper edge $\pi_t/\eta^{ch} + c^{deg}$ unless

$$c^{deg} \ge \frac{|\pi_t|}{2}\Bigl(\frac{1}{\eta^{ch}} - \eta^{dis}\Bigr),$$

so an empty band proves the idle is **not** an economic no-trade decision: no $\mu_t$ satisfies it. Measured, 14 of 59 fallback periods are empty; the 45 non-empty ones all sit far above that threshold ($c^{deg}$ of 14 to 18 against thresholds of 0.1 to 2.9), meaning wear, not price, is what holds the battery still. The check is closed-form and needs no LP, but it is **subsumed by the pinnedness test** (all 18 empty cases measured are already unpinned), so it earns its place in the reason string, not in the decision.

## Property tests

- **Soundness:** the re-solved LP objective equals the MILP objective at `1e-6` on every draw (varying $\eta$, negative prices, degradation, ramp). This is the guard on the relaxation, and the property that would have caught the `free_idle` dump.
- **Supergradient:** the fixed-commitment LP value is concave in $e_0$, so $V(e_0 + \delta) \le V(e_0) + \mu_1 \delta$ for $\delta$ of either sign. Holds under dual degeneracy, where an equality would not.
- **Marginal value:** where the MILP's two-sided finite difference in $e_0$ agrees to `1e-3` (no kink), $\mu_1$ equals it. Necessary, not sufficient: all three candidate rules pass it, which is why oracle 1 exists.
- **Band consistency, wherever a band is reported:** the reported action agrees with its own band. Charge implies $\pi_t \le \eta^{ch}(\mu_t - c^{deg})$; discharge implies $\pi_t \ge (\mu_t + c^{deg})/\eta^{dis}$; idle implies $\pi_t$ inside. This binds the dual to the dispatch it claims to explain. A band is reported at every non-fallback period and at every fallback period whose run is pinned; the invariant holds on all of them (measured 0 in 2503 non-fallback, 35 in 35 pinned fallback). It is not asserted where no band is reported.
- **A band is reported iff the run is pinned:** every unpinned run suppresses its band, every pinned run carries one. Guards the decision against being quietly widened to unpinned runs, where the band fails 43 times in 46.
- **Pinnedness is constant within a flat run:** the two fallback tie-breaks agree on $\mu$ for either all or none of a run's periods. This is what lets `pinned` live on the run.
- **Breakeven slippage is non-negative at every executed trade:** an executed charge or discharge clears its $\kappa = 0$ threshold, so it can absorb $\ge 0$ transaction cost. A negative value would mean the schedule traded at a loss against its own duals, an optimality violation.
- **Flat on interior SoC:** $\mu_t = \mu_{t+1}$ wherever $e_{\min} < e_t < e_{\max}$ (measured clean, 0 in 1147).
- **Scale invariance:** scaling capacity, both power caps, $e_0$, and $e^{tgt}$ by $k > 0$ leaves every $\mu_t$ unchanged, since the water value is a per-MWh rate. Mirrors the R1.2 property.
- **Determinism:** fixed inputs give bit-stable water values.

## Resolved decisions (reviewed 2026-07-16)

Phase-local decisions only. Each was posed as an open question with a proposed answer, then resolved in place; the resolved lines are the decision trail. Three (1, 2, and the retired rival in 1) were settled by measurement, recorded in "De-risked before drafting"; the rest are scope and interface calls. Formulation §R2.4 and implementation follow human approval of this spec.

1. **Which LP to re-solve at idle periods.**
**Resolved: `free_idle` restricted to $\pi_t \ge 0$, with $u^\star$ kept at negative-priced idle periods and objective equality asserted on every solve** (2026-07-16). Shapes ADR-0023.
    *Why this rule.* It is the only candidate measured to recover the true $\partial V^\star / \partial e_0$ on the reference instance (100, against 200 for `fix_u` and 10 for `fix_zero`, all three at the identical correct objective 190). Its one soundness failure has a closed form, is confined to negative-priced idle periods at $\eta^{rt} < 1$, and the $\pi_t \ge 0$ restriction removes it: 0 violations in 600 instances, against 2 unrestricted.
    *Why the equality assertion is not optional.* The rule works by *relaxing*, which forfeits the containment argument that makes plain fix-and-resolve sound. At $\eta^{rt} = 1$ the relaxation is provably exact (the round trip is cash-neutral and energy-neutral, so it can be removed without changing feasibility). Below 1 that proof does not extend, and no counterexample was found either, so the property is guarded rather than claimed.
    *Why not the rival, and why the rival is not merely more expensive.* Intersecting the two tie-breaks is sound but cannot be read off a solver, which returns a vertex of the dual face rather than the face; on the reference instance the two vertices are 200 and 10 and the truth is neither. Recovering the faces costs roughly $4T$ auxiliary LPs and an explicit dual. And it buys nothing where `free_idle` already applies, because `free_idle` **is** the intersection computed in one LP: each tie-break can impose only one side of the closed-form bound $\pi_t \eta^{dis} - c^{deg} \le \mu_t \le \pi_t/\eta^{ch} + c^{deg}$ (measured 0 / 205 each), while `free_idle` imposes both (0 / 203). Full evidence in "De-risked before drafting"; the rival survives as decision 2's tool, not as a rival rule.
    *What would overturn this.* A counterexample where the restricted rule's LP beats the MILP (the assertion would catch it in CI), or an instance with a $\pi_t \ge 0$ idle period whose reported $\mu_t$ misses a two-sided finite difference.
2. **What to report at a negative-priced idle period,** where the fallback leaves $\mu_t$ dependent on the solver's tie-break.
**Resolved: report $\mu_t$ everywhere; report the band if and only if the period's flat run is pinned; detect pinnedness by solving the fallback tie-break both ways and comparing $\mu$** (2026-07-16). Carried by [ADR-0023] alongside decision 1.
    *Why not the flag-only answer first proposed.* A `pinned=False` flag beside a band that contradicts the action 13 times in 21 is a wrong explanation wearing a caveat, and negative prices are a recurring BE/NL condition this project already refuses to treat as an edge case (R1.4c).
    *Why not suppress every fallback band either.* That was the fallback position and the measurement beat it. Pinnedness is a sufficient condition to report the band (35 / 35, with no counterexample in the table), so blanket suppression discards 35 of 81 fallback periods whose band is verifiably correct.
    *Why the detection is trustworthy.* It measures tie-break invariance directly rather than reasoning about it: if both sound tie-breaks agree on $\mu_t$, the tie-break did not matter, whatever the cause. It costs **one extra LP** for the horizon, against roughly $4T$ for the dual-face intersection, which is now fully retired: this buys the same safety far cheaper.
    *Why `pinned` is per-run.* $\mu$ is constant while SoC is interior, so a tie-break moves a whole flat run's level. Measured constant within every run, 110 / 110; a run holding a trading period that dispatches strictly between zero and its cap is pinned 76 / 76. The interface carries the run, not a per-period flag.
    *What the suppressed periods still get.* An empty band ($c^{deg} < |\pi_t|(1/\eta^{ch} - \eta^{dis})/2$) proves the idle is not economic, which is worth saying in the reason. It does not gate the decision, since every empty case measured is already unpinned.
    *What would overturn this.* A fallback period where both tie-breaks agree yet the band fails, which the band-consistency property would catch.
3. **Does R2.4 explain the R2.3 two-stage program?**
**Resolved: no, out of scope** (2026-07-16). R2.4 explains the deterministic dispatch only; the two-stage duals are a later phase if wanted.
    *Why.* The R2.3 first-stage variable is a day-ahead commitment coupled to per-scenario recourse by the budget $|g^{(s)} - g^{DA}| \le \rho \bar P$, and the CVaR term carries its own multipliers (the tail slacks $z_s$, the VaR level $\eta$). Its dual prices the recourse budget, a different economic object from the SoC water value, and would need its own formulation delta and its own oracles. It is not a reuse of the SoC-balance dual.
    *Why that is enough.* The master plan scopes R2.4 as the high-signal, low-effort differentiator, and its trader-facing question ("why idle through a spike") is answered entirely by the deterministic water value. The two-stage dual answers a different, more specialist question (the marginal value of relaxing the recourse budget) that nothing in the plan asks for.
    *The cost, stated plainly.* The stochastic decision is then explained through its aggregate value metrics (VSS, the frontier) but not its duals. This is a narrowing of the original Release-2 ambition, taken deliberately: the two-stage dual is a genuine later phase, not a rider on this one.
    *Consequence.* `Depends on` stays R1.1 / R1.2 / R1.5, and `bess.explain` imports only `optimizer` and `assets`, so the layering contract's `explain -> stochastic` edge stays unused-but-permitted (R2.4 could have shipped before R2.3). Reopen and add R2.3 to `Depends on` only if the two-stage dual is later brought into this phase.
4. **Slippage / transaction costs,** which the master plan files under R2.4.
**Resolved: in scope as an analysis on the band, not as an objective term** (2026-07-16). The read-off ships in R2.4; the model change does not.
    *The distinction that resolves it.* Two different things were bundled under one name. **In the objective** ($-\kappa \tau_t$ in the max) it re-optimises to a different schedule and moves objective 40 and the degradation oracles, which R2.4's gate forbids. **As analysis** it reads the schedule R1.1 already produced and its duals, and reports per trade the slippage that trade absorbs before it flips. The second changes no objective term and re-solves nothing, so it touches no R1 oracle and carries its own (oracle 7).
    *Why it belongs here.* It reuses R2.4's own product, the no-trade band, into which $\kappa$ enters directly (widening it flat by $\kappa$ each side). It is genuinely explanatory, a per-trade robustness margin, not a separate economic model, and it is cheap: a read-off, no new solve. The duration axis (ADR-0022) is precedent for folding an additive analysis into a phase rather than spinning a new one.
    *What stays out.* The objective-term version, re-optimising the dispatch under $\kappa$, is a real formulation delta with its own oracles and belongs to its own phase. Folding it here would make R2.4 violate its own no-R1-gate-weakened gate. That is the line: R2.4 explains how much cost the *computed* schedule can absorb; it does not re-plan around a cost.
5. **Endpoint shape, and its interaction with the R1.5 circuit breaker.**
**Resolved: a separate `POST /explain`, no breaker, returning schedule and explanation together; 503 if the optimal solve fails** (2026-07-16).
    *Why no breaker.* The breaker's fallback path serves a greedy schedule, a percentile heuristic with no duals at all (the same reason explaining the fallback is out of scope). A breaker-wrapped `/explain` would therefore return a null water value, band, and breakeven beside a schedule that looks authoritative, an explanation-shaped object with nothing in it.
    *Why 503 rather than a degraded 200.* The two endpoints have opposite contracts. `/dispatch` promises **availability**: it is on the trading path and a degraded-but-tradeable answer beats none, which is what the breaker guarantees. `/explain` promises **fidelity**: a hollow or wrong explanation is worse than an honest failure, so a solve that does not reach optimality surfaces as 503, not a 200 with empty fields.
    *Why return both together.* A caller wanting an explanation almost always wants the schedule it explains, so `/explain` takes the same prices + spec, solves once, and returns schedule and explanation from that one solve. This avoids a wasteful second solve and removes any risk of explaining a schedule that differs from the one returned. `/dispatch` stays the lean, breaker-protected trading endpoint; `/explain` is the richer diagnostic that always solves fresh and fails honestly.
